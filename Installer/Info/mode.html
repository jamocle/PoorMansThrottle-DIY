Motor Driver Modes

One app, multiple motor driver control styles.

The app supports several motor driver interface types so the same throttle can be configured for different real-world motor driver boards. Each mode changes the pin configuration to match a specific control scheme, helping the app work across a broad range of common hardware.

Overview

Four motor driver modes are supported: DUAL_PWM, PWM_DIR, PWM_BIDIR, and DUAL_INPT. Each mode represents a different way a motor driver expects speed and direction signals to be provided.

Control model summary
  • DUAL_PWM: two PWM pins, with direction determined by which PWM output is active.
  • PWM_DIR: one PWM pin for speed and one digital pin for direction.
  • PWM_BIDIR: one PWM enable pin plus separate forward and reverse logic pins.
  • DUAL_INPT: two input pins, with PWM applied to the active side.
Why this matters
  • The same app can be used with multiple motor driver boards.
  • Pin behavior changes to match the selected control method.
  • Setup stays aligned with the hardware actually being used.
  • The user can choose the mode that best fits current, efficiency, and driver design.

The selected motor driver mode determines how throttle, direction, and related control behavior are translated into the pin assignments shown in configuration. This provides flexibility across different driver board designs while keeping the app's control experience consistent.

Mode details and common hardware

Each mode below has its own control scheme, wiring style, and commonly supported hardware. The screenshots are included as visual references for the configuration associated with each mode.

DUAL_PWM

Separate PWM for forward and reverse

This mode uses two PWM outputs: one for forward and one for reverse. Only one side is active at a time, so direction is determined by which PWM channel is being driven.

Optional enable pins may also be used, depending on the driver board. In a typical arrangement, the forward pin is connected to the driver's forward PWM input and the reverse pin is connected to the reverse PWM input.

Common hardware: IBT-2 BTS7960 motor driver.

Characteristics: high current capability, strong fit for heavier loads, and a control model that handles direction by selecting the active PWM path.

PWM_DIR

One PWM pin plus one direction pin

This mode uses one PWM pin for speed and one digital pin for direction. The PWM signal controls speed, while the direction line selects forward or reverse.

In a typical arrangement, the PWM output is wired to the driver's PWM input and the direction output is wired to the driver's direction input. A common logic pattern is direction high for forward and low for reverse.

Common hardware: Cytron MD10C and Cytron MDD10A motor drivers.

Characteristics: clean and simple control, widely used in higher-end driver designs, with good efficiency and reliability.

PWM_BIDIR

PWM enable with forward and reverse logic pins

This mode uses one PWM pin as an enable or speed signal together with two digital pins for forward and reverse logic control.

In a typical arrangement, PWM is connected to the enable pin, while the forward and reverse pins are connected to the driver's input lines. A common logic pattern is forward high and reverse low for forward motion, reverse high and forward low for reverse motion, and both low for coast or brake depending on the driver.

Common hardware: L298N motor driver module and L293D motor driver board.

Characteristics: common and inexpensive, well suited to basic H-bridge boards, but generally less efficient and not ideal for heavier current demands.

DUAL_INPT

Two-input H-bridge style control

This mode uses two input pins, with PWM applied to whichever side is active. One side receives PWM while the other stays low, then the pattern swaps when direction changes.

In a typical arrangement, input A is wired to one driver input and input B is wired to the other. The active side receives the PWM signal that controls the motor.

Common hardware: DRV8833 motor driver module, TB6612FNG motor driver board, and MX1508 motor driver module.

Characteristics: efficient modern driver style, common on compact boards, and a good fit for low-to-medium current applications.

Recommended pairings generally follow load requirements: higher-current applications are a good fit for DUAL_PWM with boards like the IBT-2 BTS7960, or PWM_DIR with boards such as the Cytron MD10C. For medium and lighter loads, DUAL_INPT is often a strong choice with boards like the TB6612FNG or DRV8833. PWM_BIDIR remains useful and common, but boards such as the L298N and L293D are typically less suitable for heavy loads.

In summary, these modes let the app support the major DC motor driver control patterns found in hobby and embedded systems. The result is a flexible configuration model that can adapt to a wide variety of supported hardware while preserving a consistent throttle experience.
