PMT Hardware Installation

Hardware guidance for building and wiring PMT.

This page is focused on the physical side of Poor Man's Throttle: choosing a supported motor driver mode, wiring the controller and motor driver, powering the ESP32 correctly, and connecting a default headlight output. Use it as a practical installation reference while assembling a PMT throttle.

System overview

PMT is designed so the same app and controller can support several different motor driver boards. The important thing is to choose the driver mode based on the board's control pins, then wire the ESP32 to match that control style. Power for the motor driver and motor comes from the main power source, while the ESP32 is powered separately through a 5V power module feeding USB-C.

Typical power path
  • Main power source feeds the motor driver.
  • A 5V power module powers the ESP32 through USB-C.
  • The ESP32 sends control signals to the motor driver.
  • The motor driver sends power to the locomotive motor or track output.
Important hardware idea
  • Select the PMT motor driver mode by control interface, not just by product family name.
  • Match the GPIO mapping in the app to the wiring on the board.
  • Use shared ground between the ESP32 and the motor driver logic side.
  • After wiring, confirm the mode and pin mapping in the PMT app before running the motor.
For PMT hardware setup, think in three layers: power, motor driver wiring, and app configuration. When those three match, installation becomes much easier to verify and troubleshoot.

Motor driver modes and supported boards

PMT supports four motor driver control interfaces. Each one expects a different wiring pattern between the ESP32 and the motor driver board. Choose the mode that matches the board's pin behavior.

DUAL_PWM

Separate forward and reverse PWM

Uses separate forward and reverse PWM paths, typically with enable pins. This is a strong fit for higher-current applications and heavy train loads.

Typical control pins: RPWM, LPWM, R_EN, L_EN

Example boards:

  • IBT-2
  • IBT-20
  • BTS7960 modules
  • BTS7960B modules
  • BTN7960 / BTS7960 style dual-PWM drivers
  • Cytron MDD10A
  • Cytron MDD20A
  • Cytron MDD30A
  • Monster Moto Shield (VNH2SP30)
  • VNH5019 driver boards
  • VNH3SP30 modules
  • Sabertooth drivers in PWM mode
  • Roboclaw drivers in PWM mode
PWM_DIR

One PWM pin and one direction pin

Uses one PWM signal for speed and one direction signal for forward or reverse. This is a simple and clean control model used by several dedicated motor drivers.

Typical control pins: PWM, DIR

Example boards:

  • Cytron MD10C
  • Cytron MD13S
  • Cytron MD30C
  • Cytron MDS40A
PWM_BIDIR

PWM enable plus forward/reverse logic

Uses one enable or PWM input together with separate logic inputs for direction. This is common on many inexpensive H-bridge boards.

Typical control pins: EN / PWM, IN1, IN2

Example boards:

  • L298N
  • L293D
  • TB6612FNG
  • DRV8871
  • DRV8876
  • MC33926
  • MX1508
  • SN754410
DUAL_INPT

Two H-bridge inputs with PWM applied to the active side

Uses two motor-control inputs. PMT applies PWM to one side in one direction and to the other side when direction changes.

Typical control pins: IN1, IN2

Example boards:

  • DRV8833
A practical rule of thumb: DUAL_PWM is often the right place to start for high-current train installations, while PWM_BIDIR is common when using boards like the L298N. Always confirm the board's actual control pins before selecting the PMT mode in the app.

IBT-2 installation for PMT

The IBT-2 is a common PMT choice for the DUAL_PWM mode. The wiring example below shows the ESP32, the IBT-2, the motor connection, the 5V power module, and the main power source.

For an IBT-2 style PMT setup, the control wiring typically maps the ESP32 to the motor driver as follows: GPIO25 → RPWM, GPIO26 → LPWM, GPIO27 → R_EN, GPIO33 → L_EN, VIN/5V → VCC, and GND → GND. The motor driver's power side then connects to the main power source and to the motor output.

In the PMT app, configure the motor driver mode as DUAL_PWM and confirm the displayed pin mapping matches the physical wiring. This is the key checkpoint between the hardware build and the app setup.

PMT build guidance for IBT-2 installations also recommends a separate 5V logic supply for the ESP32, delivered through USB-C, while the motor driver itself is fed by the main motor power path.

L298N installation for PMT

The L298N is a common example of a PWM_BIDIR driver. The board image helps identify the major control and power pins, and the PMT wiring example shows how to connect it to the ESP32.

In a PMT L298N installation, the control pattern follows the PWM_BIDIR model. A typical mapping is GPIO25 → IN2, GPIO26 → IN1, GPIO33 → ENA, with shared ground and the 5V logic connection tied back to the ESP32 power side as shown in the wiring example.

In the PMT app, configure the motor driver mode as PWM_BIDIR and verify that the pin mapping shown in the configuration screen matches the actual board wiring you used. The board image is helpful here because it labels the enable, input, and output terminals clearly.

The L298N is widely available and easy to find, but it is generally a better fit for lighter-duty use than for heavy-current train installations. For larger loads, PMT users often prefer boards that fit the DUAL_PWM control style.

Default headlight output wiring

PMT also supports function outputs for lighting. The example below shows how to wire a simple LED to the default headlight output and how that output appears in the app configuration.

By default, the first light function is the configurable Headlight output and it is defaulted on GPIO Pin 4. This can be modified and changed freely via the app. In the app, the headlight is shown as a named output with its pin assignment, pattern, direction behavior, and linked function.

For a basic installation, wire the LED as shown in the example and then confirm in the app that the Headlight output is present on GPIO 4. This gives the user a straightforward way to add a visible front light without changing the default output assignment.

The default headlight configuration is a useful first accessory test because it validates both the physical output wiring and the app's function-output configuration at the same time.

PMT hardware checklist

Before first power-up, walk through the physical installation one more time.

Power
  • Main power source connected with correct polarity.
  • ESP32 powered from a 5V module through USB-C.
  • Ground shared correctly between control and driver logic.
  • No exposed conductors touching adjacent terminals.
Motor driver
  • Driver mode selected in PMT matches the board's control interface.
  • GPIO pin mapping in the app matches the real wiring.
  • Motor connected to the correct output terminals.
  • Direction can be corrected later by swapping motor leads if needed.
Lighting
  • Headlight output wired to the intended LED.
  • Headlight shown in the app on GPIO Pin 4 by default.
  • Pattern and direction settings reviewed in the app.
Final verification
  • IBT-2 uses DUAL_PWM configuration in the app.
  • L298N uses PWM_BIDIR configuration in the app.
  • Only supported board control styles are selected.
  • Wiring and app settings are checked together before running the motor.
The fastest way to troubleshoot PMT hardware is to compare three things side by side: the physical wiring image, the board's pin names, and the PMT configuration screen. If all three agree, the installation is usually very close.
