Auxiliary Output Support

Lighting and control outputs are now fully configurable.

The app supports up to 12 FX outputs for auxiliary control and lighting. Each output can be named, assigned to an ESP32 output GPIO pin, given a behavior pattern, linked to another output when needed, and configured as either a normal toggle or a momentary action.

What auxiliary output support includes

Auxiliary outputs are designed for much more than simple lighting. They can be used for headlights, reverse lights, secondary lights, and other non-lighting control functions where a controllable output is needed. Because each output can be individually configured, the app can support a wide range of accessory and control behaviors from the same configuration system.

Capacity

Up to 12 FX outputs

The app supports twelve separate auxiliary outputs. This gives enough room for basic headlight and reverse light behavior, additional lighting effects, and extra control outputs beyond lighting.

Per-output configuration

Each output is individually defined

Every output can have its own name, GPIO pin assignment, pattern, direction behavior, linked relationship, and interaction style. That means the outputs can be tailored to the layout and purpose of a specific throttle.

Naming

Outputs can be clearly labeled

Each output can be given a meaningful name such as Headlight, Reverse Light, 2nd Light, or a custom control label. This makes the control panel easier to understand when the output is presented to the user on the throttle screen.

GPIO assignment

Mapped to ESP32 output pins

Each output can be assigned to an ESP32 output GPIO pin. This lets the app configuration match the physical wiring of the accessory or lighting function being controlled.

Auxiliary output support is intended for both lighting and general-purpose control. A solid on/off output can be just as useful for a non-lighting accessory as it is for a lamp.

Supported output patterns

Each auxiliary output can be assigned a pattern that defines how it behaves when active.

SOLID

Simple on and off

Solid provides a straightforward on/off output. It is ideal for standard lighting such as headlights and reverse lights, and it is also useful for non-lighting auxiliary controls where a simple switched output is needed.

DBL_BLNK

Double Blink

Double Blink creates a repeating double-flash pattern. This is useful when a more noticeable flashing effect is required for a light or indicator output.

FRED

Flashing Rear End Device

FRED provides a flashing rear-end device style pattern. This is intended for marker or end-of-train style visual effects.

BLINK+

Positive phase blink

Blink+ flashes on the positive phase. It is especially useful when alternating flash behavior is desired between two related outputs.

BLINK-

Negative phase blink

Blink- flashes on the negative phase. When paired with Blink+, it can create alternating flashing behavior across linked or coordinated outputs.

Linked outputs and button behavior

Outputs can be coordinated with each other through Linked FX, and each output can also be set to operate either as a standard toggle or as a momentary control.

Linked FX

Outputs can reference another output

Each auxiliary output can be linked to another output by using Linked FX. This allows outputs to work in a coordinated way rather than completely independently.

By default, the Headlight and Reverse Light are linked, which provides a ready-made paired behavior for directional lighting.

Interaction style

Toggle or momentary

Outputs can be configured as a normal toggle style control, which is the default, or as momentary, where the output stays active only while the control is being pressed.

Momentary behavior is useful for controls that should not stay latched on after the press ends.
